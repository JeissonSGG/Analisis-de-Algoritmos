# Anlisis-de-Algoritmos
Repositorio para Análisis de Algoritmos
🔹 HouseRobber

Este código calcula cuánto dinero puedes robar como máximo sin robar casas seguidas. En cada casa decide si es mejor robarla o saltarla, usando lo que ya calculó antes.

Complejidad temporal: O(n) → revisa cada casa una sola vez.
Complejidad espacial: O(1) → solo usa unas pocas variables, no guarda listas grandes.
🔹 MaxSubarray

Encuentra la mayor suma posible de números consecutivos dentro del arreglo. En cada paso decide si continuar sumando o empezar desde cero.

Complejidad temporal: O(n) → recorre el arreglo una vez.
Complejidad espacial: O(1) → solo usa variables simples.
🔹 MinCostClimbingStairs

Calcula el menor costo para subir una escalera, pudiendo avanzar 1 o 2 pasos. En cada paso elige la opción más barata según lo anterior.

Complejidad temporal: O(n) → revisa cada escalón una vez.
Complejidad espacial: O(1) → no usa memoria extra importante.
🔹 EraseOverlapIntervals

Este código encuentra cuántos intervalos hay que eliminar para que ninguno se cruce. Primero los ordena y luego revisa cuáles se solapan.

Complejidad temporal: O(n log n) → tarda más porque primero ordena los intervalos.
Complejidad espacial: O(1) → usa pocas variables adicionales.
